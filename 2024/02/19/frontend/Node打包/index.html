CommonJS

  1. 模块同步
  2. 形成一个完整的JS文件
  3. 不会存在 require 这类的模块化语法
  4. nodejs专用

AMD

  1. 模块异步，依赖前置
  2. 加载完依赖后立即执行依赖模块
  3. 依赖加载成功后执行回调
  4. 前端专用

CMD

  1. 模块异步，延迟执行
  2. 就近依赖，先加载所有依赖模块
  3. 运行时才执行require内容，按顺序执行

ESM

  1. 遇到模块加载命令import，就会生成一个只读引用
  2. 执行时，再根据这个只读引用，到被加载的那个模块内取值
  3. 编译时就能确定模块的依赖关系，因此能够只包含要运行的代码
